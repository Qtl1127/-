# 人工确认规则

## 默认要求

每个 SPU 正式 GPT 生图前，默认需要人工确认。用户明确授权批量生成或不再逐张确认时，可以跳过生图前逐张确认，但不能跳过归档前质检和人工确认。

## 确认前必须展示

1. 当前 SPU
2. 原图路径和数量
3. 图片类型判断
4. 对应提示词文件、提示词来源和提示词摘要；若使用 `通用提示词.txt`，必须明确说明“同名提示词缺失，已采用通用提示词”
5. 输出格式、质量和尺寸
6. 是否多图统一风格
7. 不可改变项摘要
8. 即将执行的 GPT 生图任务
9. 待处理清单、缺失清单和建议执行顺序
10. 当前图片的太阳镜人物氛围方向、独立风格色调方向、`CKULH` 主视觉品牌艺术字方向，以及眼镜本体佩戴保真要求

确认话术：

```text
请确认是否执行该 SPU 的 GPT 生图升级任务。
确认后我将继续处理；未确认前不移动原图、不归档、不生成完成日志。
```

## 确认后允许

1. 生图前确认后，允许执行 GPT 生图并保存待检升级图。
2. 质检通过并完成归档前确认后，才允许移动原图、生成完成日志和标记完成。
3. 用户确认某张升级图采用后，必须立即把该 SPU 原图移动到 `02_完成后文件夹/{SPU}/原图/`，把升级图移动或保存到 `02_完成后文件夹/{SPU}/升级版/`，并生成 `处理日志.md`。
4. 没有专属 SPU 文件夹的单张图片任务，经用户确认采用后，不必单独创建文件夹；升级图统一保存到 `02_完成后文件夹/升级版/`，处理日志保存到 `02_完成后文件夹/处理日志/`，生成图保留原文件主名并按 PNG 输出。
5. 单张确认后的质检和归档动作必须在开始下一张图或下一个 SPU 前完成并复核，禁止积压多张已确认图后再统一迁移。

## 自动确认

用户写明“跟进员跟进任务”后，默认授权全 SPU 批次监督模式下的自动确认。该模式中，单个 task 质检通过后视为已确认采用，必须立即归档、清空临时文件夹、重新扫描状态并继续下一个未完成 task。无阻塞时不得只回复“下一张跟进顺序”后停止。

只有 GPT 通道失败、P0 阻塞、缺文件、质检硬性违规、临时文件清理失败，或用户明确要求暂停时，才允许停止并等待人工处理。

## 未确认前禁止

未确认前禁止移动原图、删除原图、标记完成、批量跳到下一个 SPU 或把失败图当作完成图归档。
