# 处理流程

## 1. 扫描任务

1. 扫描 `01_原图文件夹`，识别 SPU 子文件夹和直接放在根目录下的单张图片任务。
2. 在 `04_提示词文件夹` 查找同名提示词文件；同名提示词不存在时，查找并使用 `通用提示词.txt`。
3. 忽略依赖、缓存、压缩包和临时目录。
4. 正式生成前必须输出待处理清单、缺失清单和建议执行顺序。

## 2. 完整性检查

处理前必须确认：

1. SPU 原图文件夹存在且含图片；或单张图片直接存在于 `01_原图文件夹` 根目录下。
2. 同名提示词文件存在；若不存在，则 `通用提示词.txt` 必须存在。使用通用提示词时，必须在确认信息、质检记录和处理日志中说明。
3. 图片数量、尺寸要求、任务类型和是否需要统一风格已明确或可判断。

同名提示词缺失但通用提示词存在时，可以进入生图；没有专属文件夹但原图直接存在时，可以进入生图；同名提示词和通用提示词都缺失、原图缺失或图片无法识别时，不进入生图，按失败或待人工确认处理。

## 3. 判断任务类型

1. 单张原图默认主图。
2. 多张原图默认整套图。
3. SPU 下存在子文件夹时，子文件夹内图片仍属于同一 SPU，通常为细节图、详情页或组合任务，以提示词为准。
4. 同一 SPU 的全部图片必须一起考虑整体风格。
5. 直接放在 `01_原图文件夹` 根目录下的图片按独立单张主图任务处理，任务名使用原文件主名。

## 4. 准备生图任务

整理以下内容给 GPT 生图：

1. 原图
2. SPU 同名提示词；若同名提示词缺失，则使用 `通用提示词.txt`
3. 不可改变项摘要
4. 风格统一要求
5. 输出尺寸、格式和质量要求
6. 任务类型和主图/详情页/细节图判断
7. `09_太阳镜批次专项规则.md` 中的眼镜本体佩戴硬要求、参考场景和负面限制
8. `10_文案卖点排版提示词改写规则.md` 中的文案层级、卖点卡、品牌角标和底部弧形腰带改写要求

## 5. 人工确认

按 `06_人工确认规则.md` 展示确认信息。用户未确认前，不移动原图、不归档、不标记完成。

如果用户明确授权批量生成或不再逐张确认，可以继续生成；但归档前仍必须质检并获得人工确认。

单张图片或单个 SPU 的升级图一旦获得用户确认采用，必须立即完成归档：移动原图、整理升级图、生成处理日志。归档完成并复核无误后，才能开始下一个 SPU 或下一张图，禁止把多张已确认图片堆积到最后再统一迁移。

## 6. 执行升级

1. 固定 L3 升级强度。
2. 主要使用 GPT 生图。
3. 输出 PNG 高画质。
4. 严守不可改变项。
5. 同一 SPU 多图保持统一视觉体系。
6. 不同 SPU 或不同单张任务必须按产品本身独立设计，风格和色调不得相同。
7. 本次太阳镜任务每张图都必须生成合规 AI 人物佩戴或展示太阳镜，体现氛围感、高级感和轻奢感。
8. 左上角 `CKULH` 作为放大主视觉品牌艺术字处理，按产品风格变化设计，不固定套用同一个角标。

## 7. 质检

每张图生成后，立即按 `05_质检员/电商主图质检员.md` 逐张检查。该图质检没有问题后，才能进入下一张图或下一个任务；质检未通过时，按失败处理，不得归档完成。

## 7A. 跟进员批次监督模式

当用户写明“跟进员跟进任务”后，默认进入全 SPU 批次监督模式：

1. 目标是完成 `01_原图文件夹` 内全部可处理任务。
2. 每完成一个 task 后，必须连续执行：质检、确认或自动确认、归档、清空临时文件夹、重新扫描状态、继续下一个未完成 task。
3. 质检通过的生成图视为获得自动确认，可立即按归档规则处理对应原图、升级图和日志。
4. `下一张跟进顺序` 只能作为内部动作，不得作为无阻塞时的最终回复。
5. 只有 GPT 通道失败、P0 阻塞、缺文件、质检硬性违规、临时文件清理失败，或用户明确要求暂停时，才允许停止。

## 8. 归档与命名

通过质检并经人工确认后，有专属 SPU 文件夹的任务创建：

```text
02_完成后文件夹/{SPU}/原图/
02_完成后文件夹/{SPU}/升级版/
```

原图移动到 `原图/`，升级图保存到 `升级版/`。

归档必须在用户确认该图后立即执行，不得继续生成下一张后再补迁移。若归档失败，停止后续生成并记录失败原因。

升级图命名为：

```text
原文件名_升级版.png
```

没有专属 SPU 文件夹、直接放在 `01_原图文件夹` 下的单张任务，不必单独创建文件夹。生成图通过质检并经人工确认后，统一保存到：

```text
02_完成后文件夹/升级版/
```

处理日志保存到：

```text
02_完成后文件夹/处理日志/
```

生成图保留原文件主名，按 PNG 输出。

## 9. 日志

每个 SPU 或单张图片任务生成：

```text
02_完成后文件夹/{SPU}/处理日志.md
```

没有专属 SPU 文件夹的单张任务，日志保存为 `02_完成后文件夹/处理日志/{原文件主名}.md`。

日志记录输入、提示词来源、提示词内容摘要、输出路径、人工确认、质检结果、失败原因和下一步建议。
